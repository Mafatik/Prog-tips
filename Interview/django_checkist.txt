Сервисная архитектура (SOA)
    Состоит из слабосвязных распределенных компонентов, которые обмениваются данными по
    стандартным протоколам.

REST API
    Настройка над HTTP, который позволяет при помощи HTTP-запросов соединять различные части
    программ (мобильные приложения, вэб-приложения, БД и т.д.). Жестко привязан к
    HTTP-протоколу. В отличие от классического подхода передает клиенту не HTML, JS, CSS, а
    набор данных (обычно json)

Unit-test в Django
    Использует стандартную библиотеку питона unuttest. Запускаются через manage.py test.

SECRET_KEY
    Используется для генерации хэш-сумм. Генерируется автоматически при manage.py startproject.

ORM
    Маппер реляционных БД на ООП. Например, Pizza.objects.all() = SELECT * FROM Pizza. Есть
    разные методы, например create(), filter() = WHERE, и лукапы 
    Pizza.objects.(name_contains='peperoni') = WHERE name like %peperoni%, __icontaince 
    (не чувствителен к регистру), __startwith и тд.

GET POST
    Метод POST должен всегда использоваться если отправка данных приведет к внесению изменений
    в  базе данных на сервере. Применение данного метода должно повысить уровень защиты от CSRF.
    Метод GET должен применяться только для форм, действия с которыми не приводят к изменению
    базы данных (например для поисковых запросов). Кроме того, данный метод рекомендуется
    применять для создания внешних ссылок на ресурсы сайта. GET передает параметры запроса в URL
